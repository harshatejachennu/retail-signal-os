# RetailSignal OS Demo Walkthrough

## Two-Minute Demo Script

**0:00-0:20 — Project framing**

“RetailSignal OS is a research platform that turns retail/social market chatter into explainable Signal Cards. It is not a trading bot and it does not claim to predict stocks. The goal is to show the engineering and data-science workflow a serious alternative-data system needs: evidence, uncertainty, risk, validation, and clear limitations.”

**0:20-0:45 — Pipeline overview**

“The pipeline starts with Reddit-like events, resolves tickers, separates sentiment from market stance and intent, scores manipulation risk, creates Signal Cards, adds market backtests and SEC catalyst context, runs research validation and ML evaluation, then produces deterministic agent explanations for the dashboard and API.”

**0:45-1:10 — Why synthetic data is used**

“For the demo, I use deterministic synthetic data so the project runs without Reddit credentials, live SEC network access, paid APIs, or external LLM calls. Synthetic data proves the pipeline and validation mechanics work, but it is not real market evidence and I do not use it to claim predictive power.”

**1:10-1:35 — Show dashboard highlights**

“On Live Signals, each Signal Card includes direction, trust, manipulation risk, data quality, catalyst score, an explanation, and a ‘what could go wrong’ section. On Backtest Lab, outcomes are evaluated against stored market bars and SPY/QQQ benchmarks. On Research Validation, the app shows Granger-style diagnostics, lead-lag correlation, event-study summaries, negative controls, and ablations. On Signal Explanation, deterministic agents summarize bull, bear, catalyst, manipulation, backtest, ML, research, and risk evidence without hallucinating facts.”

**1:35-2:00 — Safety and technical takeaway**

“The main technical idea is point-in-time discipline: events store both `event_time` and `ingestion_time`, so historical features should only use information available at ingestion. The project avoids financial advice, real-money execution, and overclaiming. It is a portfolio-quality foundation for alternative-data research, not a stock prediction scam.”

## Dashboard Pages To Show

1. **Demo Data / Replay**
   - Shows synthetic event counts, Signal Card counts, backtest rows, research rows, ML rows, and the one-command demo pipeline.
   - Proves the demo can run offline and that synthetic data is clearly labeled.

2. **Live Signals / Demo Signal Cards**
   - Shows Signal Cards with direction, signal strength, trust score, manipulation risk, data quality, catalyst score, explanation, and “what could go wrong.”
   - Proves the system creates structured evidence artifacts rather than raw sentiment labels.

3. **Signal Explanation**
   - Shows deterministic bull, bear, catalyst, manipulation, backtest, ML, research, and risk sections.
   - Proves explanations are grounded in stored fields and do not require external LLM calls.

4. **Backtest Lab**
   - Shows forward-return summaries and SPY/QQQ-adjusted metrics.
   - Proves the project evaluates signal outcomes instead of only displaying social buzz.

5. **Research Validation**
   - Shows Granger-style summaries, lead-lag results, event-study output, negative controls, and ablation results.
   - Proves the project includes scientific validation tools and explicitly avoids causation overclaims.

6. **ML Evaluation**
   - Shows target distribution, baseline/logistic/random-forest status, feature importance when available, and walk-forward validation.
   - Proves the ML path uses labeled outcomes and chronological evaluation rather than random leakage-prone splits.

## How To Explain Synthetic Data Honestly

Use this wording:

“Synthetic data is generated by the repo to make the full workflow reproducible in a demo environment. It imitates Reddit-like events, SEC-like filings, and market bars across several scenarios. It is useful for proving that ingestion, replay, Signal Cards, backtesting, validation, ML, and explanation wiring work end-to-end. It is not evidence about real companies or real markets, and I would need real historical data and out-of-sample validation before making any research conclusions.”

## How To Explain Why This Is Not A Stock Prediction Scam

- It does not place trades or integrate with brokers.
- It does not provide buy/sell recommendations.
- It labels uncertainty, manipulation risk, data quality, and “what could go wrong.”
- It separates social sentiment from market stance and intent.
- It uses benchmark-adjusted backtests and validation diagnostics instead of cherry-picked examples.
- It includes negative controls and ablations to look for spurious relationships.
- It explicitly states that Granger-style tests do not prove true causation.
- It avoids external LLM calls, so demo explanations cannot hallucinate unsupported facts.
- It treats synthetic data as a reproducible demo tool, not as market evidence.
